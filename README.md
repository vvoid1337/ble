# BLE-Гид «Курск 1000»: Backend

FastAPI backend для мобильного BLE-гида по достопримечательностям. Android-приложение сканирует BLE-маяки, получает UUID ближайшей метки и запрашивает у этого API данные для карточки места.

## Стек

- Python
- FastAPI
- SQLAlchemy
- SQLite

SQLite используется специально, чтобы быстро поднять прототип без настройки PostgreSQL.

## Как запустить

Установить зависимости:

```powershell
python -m pip install -r requirements.txt
```

Запустить API:

```powershell
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Документация FastAPI будет доступна по адресу:

```text
http://localhost:8000/docs
```

## Адреса для Android-приложения

Если Kotlin-приложение запускается в Android Emulator, нужно использовать:

```text
http://10.0.2.2:8000
```

`localhost` внутри эмулятора указывает на сам эмулятор, а не на компьютер с backend-сервером.

Если приложение запускается на реальном телефоне в той же Wi-Fi сети, backend нужно запускать так:

```powershell
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

После этого в приложении нужно указать IP-адрес компьютера в локальной сети, например:

```text
http://192.168.1.50:8000
```

## API

### Проверка работы сервера

```http
GET /health
```

Ответ:

```json
{
  "status": "ok"
}
```

### Получить все достопримечательности

Этот эндпоинт нужен для кеширования данных в мобильном приложении, чтобы карточки могли открываться без интернета.

```http
GET /landmarks
```

Пример ответа:

```json
[
  {
    "uuid": "A1B2C3D4-E5F6-7890-ABCD-EF1234567890",
    "name": "Знаменский собор",
    "emoji": "⛪️",
    "description": "...",
    "fact": "...",
    "year": "1816",
    "public_key": ""
  }
]
```

### Получить достопримечательность по UUID маяка

```http
GET /landmark/{uuid}
```

Пример:

```text
http://localhost:8000/landmark/A1B2C3D4-E5F6-7890-ABCD-EF1234567890
```

Ответ:

```json
{
  "uuid": "A1B2C3D4-E5F6-7890-ABCD-EF1234567890",
  "name": "Знаменский собор",
  "emoji": "⛪️",
  "description": "...",
  "fact": "...",
  "year": "1816",
  "public_key": ""
}
```

Если UUID не найден, сервер вернет `404`.

## Роль backend в проекте

Backend отвечает за:

- хранение связи между UUID BLE-маяка и достопримечательностью;
- выдачу данных для карточки места в мобильном приложении;
- выдачу полного списка мест для offline-кеша;
- хранение `public_key` для защищенного режима маяков в extra-треке.

Kotlin-приложение отвечает за:

- сканирование BLE-маяков;
- фильтрацию RSSI и определение приближения к маяку;
- пользовательский интерфейс;
- локальный кеш данных;
- работу приложения-эмулятора BLE-маяка.

## Логика работы

Общий сценарий:

```text
BLE-маяк рядом с объектом
        ↓
Android-приложение находит UUID маяка
        ↓
Приложение делает запрос в FastAPI
        ↓
Backend ищет UUID в SQLite
        ↓
Приложение получает JSON и показывает карточку места
```

Пример:

```text
GET /landmark/A1B2C3D4-E5F6-7890-ABCD-EF1234567890
```

Вернет карточку:

```text
Знаменский собор
1816
Описание
Интересный факт
```

## Extra-трек по защите от подмены

Поле `public_key` уже есть в ответе API, но пока возвращается пустой строкой:

```json
{
  "public_key": ""
}
```

Позже его можно использовать для проверки подлинности маяка:

- приложение получает публичный ключ достопримечательности с backend;
- BLE-маяк или эмулятор подписывает challenge/nonce;
- приложение проверяет подпись;
- если подпись верная, карточка считается доверенной;
- если подпись неверная, приложение показывает предупреждение о возможной подмене.
